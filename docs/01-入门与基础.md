# 第一章：入门与基础

## 1.1 最简单的 Jasper 程序

```jasper
function main(): int {
    return 0;
}
```

Jasper 程序的入口是一个顶层 `function` 声明。`main` 函数返回 `int` 类型。

## 1.2 package 声明

```jasper
package org.example.myapp;

function main(): int {
    return 0;
}
```

`package` 声明包路径，用 `;` 结尾。

## 1.3 import 导入

```jasper
package org.example.myapp;

import std.io.File;              // 导入单个类型
import std.util.*;               // 按需导入（通配符）
import static math.sin;          // 导入静态成员
import static math.*;            // 导入所有静态成员

function main(): int {
    File.write("hello.txt", "你好");
    return 0;
}
```

## 1.4 from-import 导入（Jasper 特色）

```jasper
package org.example.myapp;

from std.math import sin, cos;            // 导入多个
from std.math import sin, cos as c;       // 别名导入
from "com/example/utils" import *;        // 字符串路径导入

function main(): int {
    return sin(0);
}
```

> `from` 和 `as` 是**软关键字**——在非导入语境下可以作为普通变量名使用。

## 1.5 顶层变量声明

```jasper
package demo;

var version: int = 1;           // 顶层可变变量
const appName: string = "Jasper";  // 顶层常量

function main(): int {
    return 0;
}
```

## 1.6 允许顶层语句

```jasper
var count: int = 0;
count++;                         // 顶层直接写语句

function main(): int {
    return count;
}
```

Jasper 允许在文件顶层直接写可执行语句（类似于脚本语言）。

## 1.7 注释

```jasper
// 行注释

/* 块注释 */

/**
 * 文档注释（暂时在语法层与块注释相同，
 * 语义层由 Checker 按需处理）
 */
```

## 语法对照

| 特性 | Parser 规则 | 对应代码 |
|------|-------------|---------|
| 文件入口 | `sourceFile` | `package? import* topLevelTypeDeclaration EOF` |
| package | `packageDeclaration` | `package name;` |
| import | `importDeclaration` | `import X;` / `import X.*;` / `import static X.Y;` |
| from-import | `fromImportDeclaration` | `from X import Y;` / `from X import Y as Z;` |
| 顶层类型 | `topLevelTypeDeclaration` | `class` / `interface` / `enum` / annotation type |

所有声明和语句都以 `;` 结尾。语句块用 `{ }`。
