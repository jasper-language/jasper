# 第四章：变量与声明

## 4.1 绑定语法

Jasper 的核心绑定语法是：

```jasper
name: Type
```

变量声明使用 `var` 或 `const`：

```jasper
var name: string = "Jasper";
const version: string = "1.0";
```

- `var`：绑定可重新赋值。
- `const`：绑定不可重新赋值。
- 变量使用前必须初始化。

## 4.2 类型推断

类型可以由初始化表达式推断：

```jasper
var count = 1;
const title = "Hello";
```

## 4.3 多绑定

Jasper 支持 Go 风格多返回/多赋值：

```jasper
function userInfo(): (string, int32) {
    return "Ada", 36;
}

var name, age = userInfo();
```

数量必须匹配，否则是编译错误。

普通变量声明不支持 tuple pattern：

```jasper
var (name, age) = userInfo(); // 非法
```

for-in 可以使用 tuple pattern：

```jasper
for ((k, v) in map) {
    println(k);
    println(v);
}
```

## 4.4 延迟初始化

`late` 不作为关键字语法。需要延迟初始化标记时使用注解：

```jasper
@Late
var cache: string;
```

`@Late` 的语义由 checker 或框架层定义；parser 只把它作为普通注解处理。

## 4.5 顶层变量

变量声明可以出现在文件顶层：

```jasper
package app;

var globalCount: int64 = 0;
const APP_NAME: string = "JasperApp";
```

## 4.6 局部变量

局部变量出现在函数或块内部：

```jasper
function example(): void {
    var local: int64 = 0;
    const label: string = "inner";
}
```

## 4.7 注解化扩展

导出、延迟初始化、装饰器等扩展不使用独立关键字语法，统一使用注解：

```jasper
@Export
class User { }

@Late
var cache: string;

@Decorate
function f(): void { }
```

## 4.8 语法对照

| 特性 | Parser 规则 | 示例 |
|------|-------------|------|
| 变量声明 | `variableDeclarationTail` | `var x: int64 = 1;` |
| 常量声明 | `variableDeclarationTail` | `const x: int64 = 1;` |
| 绑定 | `binding` | `name: Type` |
| 多绑定 | `variableBinding (',' variableBinding)*` | `var a, b = pair;` |
| 注解 | `annotation` | `@Late var cache: string;` |
