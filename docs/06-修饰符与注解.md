# 第六章：修饰符与注解

## 6.1 当前有效修饰符

当前 parser 接受的声明修饰符：

```antlr
modifier
    : annotation
    | Public
    | Protected
    | Private
    | Abstract
    | Static
    | Final
    ;
```

示例：

```jasper
public class User {
    protected var id: int64;
    private var name: string;

    public static function create(name: string): User {
        return new User(name);
    }

    abstract function describe(): string;
}
```

`final` 当前保留在语法中，具体语义继续评估。

访问控制：

- `public`：任意模块可见。
- `protected`：本类与子类可见。Jasper 是单继承语言，因此 protected 查找沿唯一 superclass 链向上解析；同包非子类不可见。
- `private`：当前声明作用域或当前类可见。

`protected constructor` 允许出现，用于限制构造只能由当前类或子类构造链调用。

`final` 语义：

- `final class` 禁止继承。
- `final function` 禁止 override。
- 变量绑定不可变使用 `const`，不使用 `final var` 表达。

## 6.2 不作为修饰符的关键字

以下关键字是永久保留字，但不作为有效修饰符：

```text
raw shared weak auto unique atomic pointer handle
```

`lock`、`defer`、`go`、`await` 是语句或表达式语法，不是声明修饰符。

`export` / `extern` 不作为关键字语法，也不进入保留关键字集合。需要时用注解表达。

包导出、延迟初始化和装饰器风格扩展都使用注解，不使用独立关键字语法：

```jasper
@Export
class User { }

@Late
var cache: string;

@Decorate
function f(): void { }
```

## 6.3 注解

注解使用 `@`：

```jasper
@Test
function demo(): void { }

@Route(path = "/users")
function users(): void { }
```

注解参数支持位置参数、命名参数和展开参数：

```jasper
@A(1, 2, name = "x", ...args)
```

对应 parser：

```antlr
annotation
    : '@' qualifiedName annotationArguments?
    ;
```
