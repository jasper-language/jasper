## 10.2 变量声明

核心绑定语法是：

```jasper
var name: Type = value;
const name: Type = value;
```

变量声明支持 Go 风格多绑定：

```jasper
var name, age = getUserInfo();
```

普通变量声明不支持 `var (a, b) = pair`。for-in 支持 tuple pattern：

```jasper
for ((k, v) in map) {
    println(k);
    println(v);
}
```

## 10.3 If

```jasper
if (x > 0) {
    println("positive");
} else {
    println("non-positive");
}
```

主体必须是 block。

## 10.4 Switch

```jasper
switch (code) {
    case 200 => {
        println("OK");
    }
    case 404 => {
        println("missing");
    }
    default => {
        println("unknown");
    }
}
```

规则：

- 没有 `match`。
- `switch` 必须有 `default`。
- `default` 只能出现一次，并且必须放在最后。
- `case => block` 不 fallthrough，不需要 `break`。
- 当前没有 `case if` guard。

## 10.5 循环

```jasper
while (i < 10) {
    i = i + 1;
}

do {
    poll();
} while (running);

for (var i: int32 = 0; i < 10; i = i + 1) {
    println(i);
}

for (item in items) {
    println(item);
}
```


## 10.7 Throw / Try / Catch / Finally

当前确认的异常控制流：

```jasper
throw err;

try {
    risky();
} catch (e: Error) {
    handle(e);
} finally {
    cleanup();
}
```

不引入额外错误处理语法作为第一版核心。

try-with-resources 保留，用于把资源生命周期绑定到 try 块：

```jasper
try (var f = open("file.txt")) {
    use(f);
} catch (e: Error) {
    handle(e);
}
```

`defer` 仍用于当前作用域退出时的通用清理逻辑。

## 10.8 Go / Await / Defer

`go` 是启动协程的语法糖，`await` 等待 Future，`defer` 在当前作用域结束时执行：

```jasper
go task();
var result = await future;

function readFile(path: string): string {
    var f = open(path);
    defer close(f);
    return f.read();
}
```

## 10.9 Lock

`lock` 是并发控制语句，不是修饰符。Jasper 使用 `lock`，不使用 `locked` 作为控制流关键字：

```jasper
lock (mutex) {
    criticalSection();
}
```

`locked` 如未来需要，可作为库注解或保留词另行设计；当前核心语法只确认 `lock (expr) { ... }`。
