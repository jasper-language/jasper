# 第三章：类型系统

## 3.1 设计原则

Jasper 是 GC 变量安全语言。类型系统第一版只保留能形成清晰心智模型的核心语法：

```jasper
T          // 普通非空引用/值类型
T?         // 可空类型
*T         // GC 安全指针类型
T[]        // 数组
List<T>    // 泛型
(A, B)     // tuple 类型
```

明确不支持：

```jasper
T!
T??
&T
&&T
raw T
shared T
```

`??` 只属于表达式层 null coalescing，不属于类型语法。

## 3.2 基本类型

```jasper
var a: int8 = -128;
var b: int16 = -32768;
var c: int32 = 42;
var d: int64 = 9223372036854775807;

var u: uint32 = 1;
var f: float64 = 3.14;
var ok: bool = true;
var ch: char = 'A';

var s: string = "hello";
var bytesValue: bytes = b"\x00\x01";
var pattern: regex = r"\d+";
var value: any = 42;
```

`int`、`uint` 是默认位宽语法糖，具体位宽仍待定。`float` 对应默认浮点类型，当前按 `float64` 理解。

## 3.3 可空类型

默认类型非空：

```jasper
var name: string = "Ada";
```

只有显式 `?` 类型可以为 `null`：

```jasper
var maybeName: string? = null;
```

类型层没有 `!`，因为非空是默认规则；类型层也没有 `??`。

## 3.4 指针类型

`*T` 是 GC 管理下的安全指针类型：

```jasper
var x: int32 = 1;
var p: *int32 = &x;
var y: int32 = *p;
```

规则：

- `*T` 是类型。
- `&expr` 是取地址表达式，结果类型为 `*T`。
- `*expr` 是解引用表达式，结果类型为 `T`。
- Jasper 没有 `unsafe` 语法。
- Jasper 没有 `&T` 引用类型。

## 3.5 数组

数组使用后缀 `[]`：

```jasper
var xs: int32[] = [1, 2, 3];
var matrix: int32[][] = [[1, 2], [3, 4]];
var nullableItems: string?[] = [null, "x"];
```

## 3.6 泛型

泛型类型实参使用 `<...>`：

```jasper
var names: List<string> = List<string>();
var table: Map<string, int32> = Map<string, int32>();
```

泛型声明：

```jasper
class Box<T> {
    public var value: T;
}

function first<T>(items: T[]): T? {
    return items[0];
}
```

泛型约束使用 `extends` 或 `where`：

```jasper
function max<T extends Comparable<T>>(a: T, b: T): T {
    return a;
}

function sort<T>(items: T[]): T[]
where T is Comparable {
    return items;
}
```

## 3.7 Tuple

Tuple 类型使用括号和逗号：

```jasper
var pair: (string, int32) = ("Ada", 36);
var nested: (string, (int32, bool)) = ("x", (1, true));
```

括号分组和 tuple 的区别：

```jasper
(a)       // 分组，不是 tuple
(a, b)    // tuple
(a,)      // 非法：第一版不支持单元素 tuple
```

Tuple 可用于多值返回：

```jasper
function userInfo(): (string, int32) {
    return "Ada", 36;
}
```

Tuple 访问使用索引：

```jasper
var pair: (string, int32) = ("Ada", 36);
var name: string = pair[0];
var age: int32 = pair[1];
```

普通变量声明不使用 `var (a, b) = pair` 形式；多返回/多赋值使用 `var a, b = pair`。

## 3.8 Any 与泛型通配

`any` 是顶层类型，也用于泛型通配：

```jasper
var values: List<any> = List<any>();
var animals: List<any extends Animal> = getAnimals();
var sinks: List<any super Dog> = getDogSinks();
```

`?` 只用于 nullable，不用于泛型 wildcard。

## 3.9 保留关键字

以下关键字永久保留但不实现：

```text
raw shared weak auto unique atomic pointer handle
```

它们不能用于变量名、类型名或修饰符，也不进入类型规则。

## 3.10 语法对照

| 特性 | Parser 规则 | 示例 |
|------|-------------|------|
| 基本类型 | `primitiveType` | `int32`, `float64`, `bool` |
| 命名类型 | `typeAtomBase` | `User`, `List<T>` |
| 可空类型 | `typeQualifier` | `string?` |
| 指针类型 | `typePrefix` | `*int32`, `**Node` |
| 数组类型 | `typeSuffix` | `int32[]` |
| 泛型实参 | `typeArguments` | `Map<string, int32>` |
| 泛型通配 | `wildcard` | `any`, `any extends T`, `any super T` |
| Tuple | `typeAtomBase` | `(string, int32)` |
